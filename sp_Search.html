Project: sp_Search

A powerful and flexible search utility for Microsoft SQL Server.

Project Goal

`sp_Search` is a stored procedure designed to drastically reduce the time and effort required for database developers and administrators to locate database objects, code snippets, configurations, and permissions within complex SQL Server environments.

Technologies Used

  • Language: T-SQL (Microsoft SQL Server)
  • Techniques: Dynamic SQL, System View Interrogation
  • Compatibility: SQL Server 2016 and higher

Key Features

  • Unified Search: Scans across multiple databases from a single execution.
  • SQL Agent Integration: Searches not only object definitions but also the names, steps, and commands within SQL Agent Jobs.
  • Targeted Search Modes: Provides specific modes to find columns, indexes, permissions, object references, active query stats, partitions, and replication configurations.
  • Advanced Control: Offers parameters to filter by database, exclude system objects, and apply custom sorting or post-filtering to the results.

Getting Started

The procedure is designed to be self-documenting. You can download the script, install it in your `master` database, and run the following command for a full tutorial: 

exec sp_Search;
